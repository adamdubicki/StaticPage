    Stock Market Prediction

    (2017)

    I was involved in a natural language processing project to predict the Dow Jones Industrial Average (DJIA). To accomplish this, Reddit news headlines were data mined to determine whether world news headlines could be correlated with the rise and fall of the Dow Jones.

    Our data set consisted of 8 years of Reddit data, each day having 25 headlines and whether the DJIA fell or rose on that day. Other features such as the sentiment of the headlines and other preprocessing steps were added using the python library Sci-Kit-Learn.

    Our best result was a 58.8% accuracy using logistic regression, bi-grams, trigrams, and a day offset. For more details on the project, read the full report or visit the repository on github.

    BattleSnake 2

    (2017)

    BattleSnake is an annual AI coding competition in Victoria BC. It is based of the classic game of 'Snake' with a few extra rules. There are multiple snakes, and a possibility to starve if a snake does not eat within 100 turns.

    My first year of BattleSnake (2016), was a huge learning experience for my team. Only one team member had ever competed in BattleSnake before. In the end our snake was pretty simple, and only contained code for it not to eat itself. Despite this, it was almost able to move to the 2nd round of the advanced division. The repository for the first snake (2016) can be found here.

    For my second year of BattleSnake I ended up coding a more sophisticated snake. First the snake tries to determine a good piece of food to eat. A good piece of food is determined if our snake is closest to it than all other competing snakes.

    If such a piece of food exists, it finds the shortest path using A* search. If a path exists, we check to make sure that if we were to move our snake to that piece of food that we can still reach our tail. This acts a heuristic to prevent our snake from running into dead ends.

    If no path exists to a good food, our snake then tries to loop and follow its tail. This allows the snake to stall indefintely (well, until it starves to death).

    Lastly, if no good food exists, nor a path to loop, our snake tries to find the most open space available using a flood fill algorithm Below is a video of our snake in action, playing a game with itself.

    Our snake was unable to make it to the second round of the advanced division. Yet, it still gave a strong showing and was cited as a "pretty tough" by one of our opponents (a developer at RedBrick practiced against it).

    The repository for my snake (2017) can be found here.

    Bounded Buffer

    (2017)

    Bounded buffer is an interesting mutual exclusion problem. Essentially there exists a circular buffer which can hold N items. Clients can then either remove or add items to the buffer asynchronously. While conceptually simple, there exist interesting edge cases.

    The buffer has a limited size, so adding items to a full buffer should cause clients to queue up, waiting for an open space to place their item. The opposite is also true for removing items from an empty buffer. Clients wanting to remove from an empty buffer will queue up waiting for items to be added.

    For further complication, clients cannot remove the same item, nor add to the same buffer space.

    To solve this problem, two thread conditionals are used with a mutex. The two conditionals determine whether the buffer is full, or empty. When the buffer is no longer empty, it signals waiting clients that they are able to remove and vice versa for adding. The mutex acts as a guard for interacting with the buffer itself. For example, if two clients try to remove at the same time, they could accidently both try to remove the same item. This mutex gates the clients to removing items one at a time.

    For extra work, a round robin scheduler was added so that threads would be given an equal opportunity to complete their task.

    The repository for the bounded buffer problem can be found here.
